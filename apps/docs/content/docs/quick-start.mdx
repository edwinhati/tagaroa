# Quick Start

Get the Tagaroa platform running on your local machine in just a few minutes.

## Prerequisites

Before you begin, ensure you have:

- **Bun** 1.3.5 or higher
- **Docker** and Docker Compose
- **Git**
- **Node.js** 18 or higher (for compatibility)

## Installation

### 1. Clone the Repository

```bash
git clone https://github.com/your-org/tagaroa.git
cd tagaroa
```

### 2. Install Dependencies

```bash
bun install
```

This will install all dependencies for apps, servers, and packages.

### 3. Setup Environment Files

```bash
make env-setup
```

Or manually copy environment files:

```bash
cp apps/auth/.env.example apps/auth/.env
cp apps/admin/.env.example apps/admin/.env
cp apps/finance/.env.example apps/finance/.env
cp apps/web/.env.example apps/web/.env
cp servers/auth/.env.example servers/auth/.env
cp servers/finance/.env.example servers/finance/.env
```

### 4. Start Infrastructure Services

```bash
make docker-up
```

This starts:
- PostgreSQL (port 5432)
- Kafka (port 29092)
- MinIO (ports 9000, 9001)
- n8n (port 5678)
- Traefik (ports 8080, 8000)

### 5. Initialize Databases

```bash
make db-init
```

This creates databases for:
- auth
- finance
- files
- n8n

### 6. Run Database Migrations

```bash
# Auth server
cd servers/auth
bunx drizzle-kit migrate
cd ../..

# Finance server (if needed)
cd servers/finance
# Run migrations if you have them
cd ../..
```

### 7. Start Development Servers

```bash
# Start all apps and servers
bun run dev
```

This starts:
- Web app (http://localhost:3000)
- Auth app (http://localhost:3001)
- Admin app (http://localhost:3002)
- Finance app (http://localhost:3004)
- Docs app (http://localhost:3010 - configured in package.json)

### 8. Start Backend Services

In separate terminals:

```bash
# Auth server (TypeScript/Bun)
cd servers/auth
bun run dev

# Storage server (TypeScript/Bun)
cd servers/storage
bun run dev

# Finance server (Go)
cd servers/finance
go run ./cmd/main.go

# File server (Go - optional)
cd file
go run ./cmd/main.go
```

## Common Commands

### Development

```bash
# Run all apps
bun run dev

# Run specific app
turbo dev --filter=web

# Run specific server
cd servers/auth && bun run dev
```

### Building

```bash
# Build all
make build

# Build specific app
turbo build --filter=web

# Build specific server
make build-server SERVER=finance
```

### Testing

```bash
# Run all tests
make test

# Run specific tests
cd servers/auth && bun test
cd servers/finance && go test ./...
```

### Linting

```bash
# Lint all code
make lint

# Lint specific workspace
turbo lint --filter=web
```

### Docker

```bash
# Start services
make docker-up

# Stop services
make docker-down

# View logs
make docker-logs

# Build Docker images
make docker-build
```

## Troubleshooting

### Port Already in Use

If you get a "port already in use" error:

```bash
# Find process using port
lsof -i :3000

# Kill process
kill -9 <PID>
```

### Database Connection Failed

```bash
# Check PostgreSQL is running
docker compose ps postgres

# Restart PostgreSQL
docker compose restart postgres
```

### Dependencies Not Installing

```bash
# Clear cache and reinstall
rm -rf node_modules
rm bun.lock
bun install
```

### Build Failures

```bash
# Clear Turborepo cache
turbo clean

# Clear Next.js cache
rm -rf apps/*/.next

# Rebuild
turbo build --force
```

## Development Tips

1. **Use Turborepo**: Leverage caching with `turbo` commands
2. **Hot Reload**: Changes auto-reload in development
3. **Check Logs**: Use `docker compose logs -f` for infrastructure logs
4. **Database GUI**: Use Drizzle Studio for database management
5. **API Testing**: Use Traefik dashboard to test API routes
