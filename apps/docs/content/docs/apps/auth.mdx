# Auth App

Dedicated authentication application handling user sign-in, sign-up, and OAuth consent flows.

## Overview

- **Port**: 3001
- **Framework**: Next.js 16 with App Router
- **Purpose**: User authentication, OAuth flows, consent management

## Features

- User sign-in and sign-up
- OAuth consent flow
- Social authentication (Google, GitHub)
- Session management
- Better Auth integration
- Form validation with React Hook Form and Zod

### `/sign-in`
User sign-in page with email/password and social login options.

### `/consent`
OAuth consent page for third-party application authorization.

### `/`
Landing page with authentication status.

## Authentication Flow

1. User navigates to `/sign-in`
2. Enters credentials or chooses social login
3. Form validates input using Zod schema
4. Request sent to auth server
5. On success, session created and user redirected
6. Session managed via Better Auth

## Components

### Sign-in Form
- Email/password input fields
- Social login buttons (Google, GitHub)
- Form validation
- Error handling
- Loading states

## Integration with Auth Server

The auth app communicates with the auth server (`servers/auth`) via API proxy:

```typescript
// src/proxy.ts
export const authProxy = createProxy({
  baseURL: process.env.NEXT_PUBLIC_API_URL,
  prefix: '/api/auth'
});
```

## Security Features

- CSRF protection
- Secure session cookies
- OAuth 2.0 compliance
- Input validation and sanitization
- Rate limiting (server-side)

## Customization

### Adding OAuth Providers

Configure additional OAuth providers in the auth server and add buttons to the sign-in form:

```tsx
// src/components/sign-in-form.tsx
<Button onClick={() => signIn.social({ provider: 'github' })}>
  Sign in with GitHub
</Button>
```

### Styling
Customize the authentication UI using Tailwind classes and shared components from `@repo/common`.
