# Admin App

Administrative dashboard for managing users, viewing analytics, and system administration.

## Overview

- **Port**: 3002
- **Framework**: Next.js 16 with App Router
- **Purpose**: User management, system administration, analytics

## Features

- User management (CRUD operations)
- User data table with sorting and filtering
- User detail sheets
- Create user dialog
- Role-based access control
- Analytics integration
- Protected routes

## Authentication

The admin app requires authentication. Users are redirected to the auth app if not logged in.

```typescript
// Protected route example
import { auth } from '@repo/common/lib/auth-client';

export default async function AdminPage() {
  const session = await auth();
  
  if (!session) {
    redirect('/auth/sign-in');
  }
  
  // Render admin content
}
```

## API Integration

The admin app communicates with backend services:

```typescript
// Fetch users
const { data: users } = useQuery({
  queryKey: ['users'],
  queryFn: () => fetch('/api/users').then(r => r.json())
});

// Create user
const createUser = useMutation({
  mutationFn: (user) => fetch('/api/users', {
    method: 'POST',
    body: JSON.stringify(user)
  })
});
```
