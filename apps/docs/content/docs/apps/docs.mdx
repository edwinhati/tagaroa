# Docs App

This documentation site, built with Fumadocs for comprehensive platform documentation.

## Overview

- **Port**: 3010
- **Framework**: Next.js 16 with Fumadocs
- **Purpose**: Platform documentation, guides, API references

## Features

- MDX-based documentation
- Full-text search
- Syntax highlighting
- Component showcase
- Responsive design
- Dark mode support
- Auto-generated navigation

## Project Structure

```
apps/docs/
├── content/

## Writing Documentation

### Creating New Pages

Create MDX files in `content/docs/`:

```mdx
---
title: Page Title
description: Page description for SEO
---

# Page Title

Your content here...

## Section

More content...
```

### Using Components

Fumadocs provides built-in components:

```mdx
## Cards

<Cards>
  <Card title="Title" href="/link" />
  <Card title="Another" href="/another" />
</Cards>

## Code Blocks

```js
console.log('Hello World');
```

## Callouts

<Callout type="info">
  Important information
</Callout>
```

### Frontmatter Schema

Configure frontmatter in `source.config.ts`:

```typescript
export default defineConfig({
  frontmatter: {
    title: { type: 'string', required: true },
    description: { type: 'string' },
    date: { type: 'date' }
  }
});
```

## Configuration

### Source Configuration

```typescript
// source.config.ts
import { defineConfig } from 'fumadocs-mdx/config';

export default defineConfig({
  generateManifest: true,
  lastModifiedTime: 'git',
  mdxOptions: {
    remarkPlugins: [],
    rehypePlugins: []
  }
});
```

### Layout Configuration

```typescript
// lib/layout.shared.tsx
export const layoutOptions = {
  nav: {
    title: 'Tagaroa Docs',
    url: '/'
  },
  links: [
    { text: 'Documentation', url: '/docs' },
    { text: 'API', url: '/api' }
  ]
};
```

## Search

Full-text search is powered by Fumadocs:

```typescript
// app/api/search/route.ts
import { createSearchAPI } from 'fumadocs-core/search/server';
import { source } from '@/lib/source';

export const { GET } = createSearchAPI('advanced', {
  indexes: source.getPages().map(page => ({
    title: page.data.title,
    description: page.data.description,
    url: page.url,
    content: page.data.exports.content
  }))
});
```

## Customization

### Theming

Customize colors and styles in `tailwind.config.ts`:

```typescript
export default {
  theme: {
    extend: {
      colors: {
        primary: { /* ... */ },
        secondary: { /* ... */ }
      }
    }
  }
};
```

### Navigation

Navigation is auto-generated from file structure. Control order with frontmatter:

```mdx
---
title: Page Title
order: 1
---
```

### Custom Components

Add custom MDX components:

```typescript
// mdx-components.tsx
export function useMDXComponents(components) {
  return {
    ...components,
    CustomComponent: MyCustomComponent
  };
}
```

## Best Practices

1. **Organize by Feature**: Group related docs in folders
2. **Use Descriptive Titles**: Clear, searchable titles
3. **Add Descriptions**: SEO-friendly descriptions
4. **Link Between Pages**: Use relative links
5. **Code Examples**: Include working code samples
6. **Keep Updated**: Regular documentation updates
