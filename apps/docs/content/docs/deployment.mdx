# Deployment

Complete guide for deploying the Tagaroa platform to production environments.

## Deployment Options

- **Vercel**: Frontend applications (Next.js apps)
- **Docker**: Containerized deployment with Docker Compose
- **Kubernetes**: Orchestrated deployment with Flux CD
- **Bare Metal**: Direct server deployment with MetalLB and ingress-nginx

## Building for Production

### Build All

```bash
# Build all apps and servers
make build

# Or using Turborepo for frontend only
turbo build

# Or using Bun
bun run build
```

### Build Specific Components

```bash
# Build specific frontend app
turbo build --filter=web
turbo build --filter=auth
turbo build --filter=admin
turbo build --filter=finance
turbo build --filter=docs

# Build specific Go server
make build-server SERVER=finance

# Build all Docker images
make docker-build

# Build specific Docker images
make docker-build-app APP=web
make docker-build-server SERVER=finance
```

## Frontend Deployment

### Vercel (Recommended)

#### Deploy All Apps

```bash
# Install Vercel CLI
bun add -g vercel

# Login
vercel login

# Deploy
vercel deploy --prod
```

#### Deploy Specific App

```bash
cd apps/web
vercel deploy --prod
```

#### Environment Variables

Configure in Vercel dashboard or CLI:

```bash
vercel env add NEXT_PUBLIC_API_URL production
vercel env add NEXT_PUBLIC_POSTHOG_KEY production
```

#### Vercel Configuration

```json
// vercel.json
{
```

```bash

make docker-build-app APP=web
```


### Kubernetes

#### Prerequisites

```bash
# Install kubectl
brew install kubectl

# Configure cluster access
kubectl config use-context production
```

#### Deploy Services

```bash
# Deploy all services
kubectl apply -f infra/k8s/

# Deploy specific service
kubectl apply -f infra/k8s/auth-server/

# Check deployments
kubectl get deployments
kubectl get pods
kubectl get services
```

#### Example Deployment

```yaml
# infra/k8s/auth-server/deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: auth-server
  namespace: production
spec:
  replicas: 3
  selector:
    matchLabels:
      app: auth-server
  template:
    metadata:
      labels:
        app: auth-server
    spec:
      containers:
      - name: auth-server
        image: registry.gitlab.com/tagaroa/auth-server:latest
        ports:
        - containerPort: 8081
        env:
        - name: PORT
          value: "8081"
        - name: DATABASE_URL
          valueFrom:
            secretKeyRef:
              name: auth-secrets
              key: database-url
        - name: BETTER_AUTH_SECRET
          valueFrom:
            secretKeyRef:
              name: auth-secrets
              key: auth-secret
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
        livenessProbe:
          httpGet:
            path: /api/auth/health
            port: 8081
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /api/auth/health
            port: 8081
          initialDelaySeconds: 5
          periodSeconds: 5
---
apiVersion: v1
kind: Service
metadata:
  name: auth-server
  namespace: production
spec:
  selector:
    app: auth-server
  ports:
  - port: 8081
    targetPort: 8081
  type: ClusterIP
```

#### Secrets Management

```bash
# Create secrets
kubectl create secret generic auth-secrets \
  --from-literal=database-url="postgresql://..." \
  --from-literal=auth-secret="..." \
  --namespace=production

# Update secrets
kubectl delete secret auth-secrets --namespace=production
kubectl create secret generic auth-secrets \
  --from-literal=database-url="postgresql://..." \
  --namespace=production
```

#### Ingress Configuration

```yaml
# infra/k8s/ingress.yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: api-ingress
  namespace: production
  annotations:
    cert-manager.io/cluster-issuer: letsencrypt-prod
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
spec:
  ingressClassName: nginx
  tls:
  - hosts:
    - api.tagaroa.com
    secretName: api-tls
  rules:
  - host: api.tagaroa.com
    http:
      paths:
      - path: /api/auth
        pathType: Prefix
        backend:
          service:
            name: auth-server
            port:
              number: 8081
      - path: /api/finance
        pathType: Prefix
        backend:
          service:
            name: finance-server
            port:
              number: 8082
```

### Bare Metal

#### Build Binaries

```bash
# Go servers
cd servers/finance
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 \
  go build -a -installsuffix cgo \
  -ldflags '-extldflags "-static"' \
  -o finance-server ./cmd/main.go

# Rust servers
cd servers/investment
cargo build --release --target x86_64-unknown-linux-musl
```

#### Deploy with systemd

```ini
# /etc/systemd/system/auth-server.service
[Unit]
Description=Auth Server
After=network.target postgresql.service

[Service]
Type=simple
User=tagaroa
WorkingDirectory=/opt/tagaroa/servers/auth
EnvironmentFile=/opt/tagaroa/servers/auth/.env
ExecStart=/usr/bin/bun run /opt/tagaroa/servers/auth/src/index.ts
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
# Enable and start service
sudo systemctl enable auth-server
sudo systemctl start auth-server

# Check status
sudo systemctl status auth-server

# View logs
sudo journalctl -u auth-server -f
```

## Database Deployment

### PostgreSQL

#### Managed Services

- **AWS RDS**: Fully managed PostgreSQL
- **Google Cloud SQL**: Managed PostgreSQL
- **DigitalOcean Managed Databases**: Simple managed PostgreSQL
- **Supabase**: PostgreSQL with additional features

#### Self-Hosted

```bash
# Docker
docker run -d \
  --name postgres \
  -e POSTGRES_PASSWORD=secure-password \
  -v postgres-data:/var/lib/postgresql/data \
  -p 5432:5432 \
  postgres:alpine

# Kubernetes
kubectl apply -f infra/k8s/postgres/
```

#### Run Migrations

```bash
# Auth server
cd servers/auth
DATABASE_URL="postgresql://..." bunx drizzle-kit migrate

# Finance server
cd servers/finance
migrate -path db/migrations \
  -database "postgresql://..." up

# Investment server
cd servers/investment
DATABASE_URL="postgresql://..." sqlx migrate run
```

## CI/CD

### GitHub Actions

```yaml
# .github/workflows/deploy.yml
name: Deploy

on:
  push:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - uses: oven-sh/setup-bun@v1
        with:
          bun-version: 1.3.5
      
      - name: Install dependencies
        run: bun install
      
      - name: Build
        run: turbo build
      
      - name: Build Docker images
        run: make docker-build
      
      - name: Push to registry
        run: |
          echo ${{ secrets.REGISTRY_PASSWORD }} | docker login -u ${{ secrets.REGISTRY_USERNAME }} --password-stdin
          docker push registry.gitlab.com/tagaroa/auth-server:latest
          docker push registry.gitlab.com/tagaroa/finance-server:latest
      
      - name: Deploy to Kubernetes
        run: |
          kubectl config use-context production
          kubectl apply -f infra/k8s/
          kubectl rollout restart deployment/auth-server
          kubectl rollout restart deployment/finance-server
```

## Rollback

### Kubernetes

```bash
# Rollback deployment
kubectl rollout undo deployment/auth-server

# Rollback to specific revision
kubectl rollout undo deployment/auth-server --to-revision=2

# Check rollout history
kubectl rollout history deployment/auth-server
```

### Docker

```bash
# Stop current container
docker stop auth-server

# Start previous version
docker run -d \
  --name auth-server \
  tagaroa/auth-server:previous-tag
```

## Best Practices

1. **Use Environment Variables**: Never hardcode secrets
2. **Health Checks**: Implement health endpoints
3. **Graceful Shutdown**: Handle SIGTERM properly
4. **Resource Limits**: Set CPU and memory limits
5. **Horizontal Scaling**: Design for multiple replicas
6. **Database Migrations**: Run before deployment
7. **Zero Downtime**: Use rolling updates
8. **Monitoring**: Monitor all services
9. **Backups**: Regular automated backups
10. **Security**: Use HTTPS, secrets management, network policies
