# File Server

A Go-based microservice for handling file uploads, downloads, and management using MinIO as the storage backend.

## Overview

The file server provides:
- File upload with multipart form support
- File download with presigned URLs
- File listing with pagination
- Soft delete functionality
- MinIO integration for S3-compatible object storage
- PostgreSQL for metadata storage
- OIDC authentication

## Technology Stack

- **Language**: Go 1.24
- **Framework**: Custom HTTP server
- **Database**: PostgreSQL
- **Storage**: MinIO (S3-compatible)
- **Authentication**: OIDC (OpenID Connect)

## API Endpoints

### Files

- `POST /api/v1/files/upload` - Upload a file
- `GET /api/v1/files` - List files (with pagination)
- `GET /api/v1/files/:id` - Get file metadata
- `GET /api/v1/files/:id/download` - Download file
- `GET /api/v1/files/:id/url` - Get presigned URL
- `DELETE /api/v1/files/:id` - Delete file (soft delete)

### Health

- `GET /api/v1/health` - Health check endpoint

## Configuration

Environment variables (`.env`):

```bash
# Server
ENV=local                    # Environment (local, development, production)
PORT=8085                    # Server port

# Database
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=postgres
DB_NAME=files

# Storage (MinIO)
S3_ENDPOINT=localhost:9000
S3_ACCESS_KEY=minioadmin
S3_SECRET_KEY=minioadmin
S3_BUCKET=files
S3_USE_SSL=false

# Authentication (OIDC)
OIDC_BASE_URL=http://localhost:8080/api/auth
OIDC_CLIENT_ID=your-client-id
OIDC_CLIENT_SECRET=your-client-secret
```

## Usage Examples

### Upload a File

```bash
curl -X POST \
  http://localhost:8085/api/v1/files/upload \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@/path/to/your/file.pdf"
```

Response:
```json
{
  "id": "file-uuid",
  "user_id": "user-uuid",
  "filename": "file.pdf",
  "size": 1024,
  "content_type": "application/pdf",
  "storage_path": "user-uuid/file-uuid",
  "created_at": "2024-01-01T00:00:00Z"
}
```

### List Files

```bash
curl http://localhost:8085/api/v1/files?limit=10&offset=0 \
  -H "Authorization: Bearer YOUR_TOKEN"
```

Response:
```json
{
  "files": [...],
  "total": 100,
  "limit": 10,
  "offset": 0
}
```

### Get File Metadata

```bash
curl http://localhost:8085/api/v1/files/{file-id} \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### Download File

```bash
curl -O http://localhost:8085/api/v1/files/{file-id}/download \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### Get Presigned URL

```bash
curl http://localhost:8085/api/v1/files/{file-id}/url?expiry=1h \
  -H "Authorization: Bearer YOUR_TOKEN"
```

Response:
```json
{
  "url": "https://minio:9000/files/...",
  "expires_at": "2024-01-01T01:00:00Z"
}
```

### Delete File

```bash
curl -X DELETE http://localhost:8085/api/v1/files/{file-id} \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## Architecture

### Layers

```
┌─────────────────────────────────────┐
│         HTTP Handlers               │
├─────────────────────────────────────┤
│         Service Layer               │
├─────────────────────────────────────┤
│    Repository Layer (PostgreSQL)    │
├─────────────────────────────────────┤
│     Storage Layer (MinIO)           │
└─────────────────────────────────────┘
```

### Storage Layer

```go
type Storage interface {
    Upload(ctx context.Context, path string, file io.Reader, size int64, contentType string) error
    Download(ctx context.Context, path string) (io.ReadCloser, error)
    GetPresignedURL(ctx context.Context, path string, expiry time.Duration) (string, error)
    Delete(ctx context.Context, path string) error
}
```

### Repository Layer

```go
type FileRepository interface {
    Create(ctx context.Context, file *File) error
    GetByID(ctx context.Context, id string) (*File, error)
    GetByUserID(ctx context.Context, userID string, limit, offset int) ([]*File, int, error)
    Update(ctx context.Context, file *File) error
    SoftDelete(ctx context.Context, id string) error
}
```

## Database Schema

```sql
CREATE TABLE files (
    id UUID PRIMARY KEY,
    user_id VARCHAR(255) NOT NULL,
    filename VARCHAR(255) NOT NULL,
    size BIGINT NOT NULL,
    content_type VARCHAR(100),
    storage_path VARCHAR(500) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMP NULL,
    INDEX idx_user_id (user_id),
    INDEX idx_deleted_at (deleted_at)
);
```

## Best Practices

1. **Validate File Types**: Check content type and file extensions
2. **Limit File Size**: Set maximum upload size
3. **Scan for Malware**: Integrate virus scanning
4. **Use Presigned URLs**: For direct client downloads
5. **Implement Rate Limiting**: Prevent abuse
6. **Soft Delete**: Keep metadata for audit trails
7. **Monitor Storage**: Track storage usage and costs
8. **Backup Files**: Regular backups of MinIO data
