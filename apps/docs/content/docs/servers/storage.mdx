# Storage Server

Microservice for handling file uploads, downloads, and management using MinIO as the storage backend.

## Overview

- **Port**: 8084 (configurable)
- **Technology**: TypeScript with Hono and Bun
- **Storage**: MinIO (S3-compatible)
- **Database**: PostgreSQL with Drizzle ORM (metadata)
- **Purpose**: File storage metadata and management

## Features

- File metadata management
- Integration with MinIO for object storage
- PostgreSQL for metadata storage with Drizzle ORM
- Better Auth integration for authentication
- Health check endpoint
- TypeScript for type safety
- Bun for fast runtime and package management

## Architecture

### Storage Layer

```go
// internal/storage/minio.go
type MinIOStorage struct {
    client *minio.Client
    bucket string
}

func (s *MinIOStorage) Upload(ctx context.Context, path string, reader io.Reader, size int64, contentType string) error {
    _, err := s.client.PutObject(ctx, s.bucket, path, reader, size, minio.PutObjectOptions{
        ContentType: contentType,
    })
    return err
}

func (s *MinIOStorage) Download(ctx context.Context, path string) (*minio.Object, error) {
    return s.client.GetObject(ctx, s.bucket, path, minio.GetObjectOptions{})
}

func (s *MinIOStorage) GetPresignedURL(ctx context.Context, path string, expiry time.Duration) (string, error) {
    return s.client.PresignedGetObject(ctx, s.bucket, path, expiry, nil)
}

func (s *MinIOStorage) Delete(ctx context.Context, path string) error {
    return s.client.RemoveObject(ctx, s.bucket, path, minio.RemoveObjectOptions{})
}
```

### Repository Layer

```go
// internal/repository/file_repository.go
type FileRepository interface {
    Create(ctx context.Context, file *File) error
    GetByID(ctx context.Context, id string) (*File, error)
    GetByUserID(ctx context.Context, userID string, limit, offset int) ([]*File, int, error)
    Update(ctx context.Context, file *File) error
    SoftDelete(ctx context.Context, id string) error
}
```

### Service Layer

```go
// internal/service/file_service.go
type FileService struct {
    repo    FileRepository
    storage Storage
}

func (s *FileService) UploadFile(ctx context.Context, userID string, file multipart.File, header *multipart.FileHeader) (*File, error) {
    // Generate unique file ID
    fileID := generateFileID()
    
    // Create storage path
    storagePath := fmt.Sprintf("%s/%s", userID, fileID)
    
    // Upload to MinIO
    err := s.storage.Upload(ctx, storagePath, file, header.Size, header.Header.Get("Content-Type"))
    if err != nil {
        return nil, err
    }
    
    // Save metadata to database
    fileMetadata := &File{
        ID:           fileID,
        UserID:       userID,
        FileName:     fileID,
        OriginalName: header.Filename,
        MimeType:     header.Header.Get("Content-Type"),
        Size:         header.Size,
        StoragePath:  storagePath,
        CreatedAt:    time.Now(),
        UpdatedAt:    time.Now(),
    }
    
    err = s.repo.Create(ctx, fileMetadata)
    if err != nil {
        // Rollback: delete from storage
        s.storage.Delete(ctx, storagePath)
        return nil, err
    }
    
    return fileMetadata, nil
}
```

## Best Practices

1. **File Naming**: Use UUIDs for unique file names
2. **Soft Delete**: Keep metadata for recovery
3. **Presigned URLs**: Use for direct client access
4. **Chunked Upload**: For large files
5. **Cleanup**: Periodic cleanup of deleted files
6. **Backup**: Regular MinIO backups
