# Development Guide

Complete guide for setting up your development environment and contributing to the Tagaroa platform.

## Prerequisites

### Required Tools

- **Bun**: 1.3.5+ (package manager and runtime)
- **Node.js**: 18+ (for compatibility)
- **Go**: 1.24+ (for Go services)
- **Docker**: Latest with Docker Compose (for infrastructure)
- **Git**: Latest

### Optional Tools

- **Make**: For using Makefile commands (recommended)
- **kubectl**: For Kubernetes development
- **PostgreSQL Client (psql)**: For database management
- **MinIO Client (mc)**: For object storage management
- **Delve**: For Go debugging
- **Rust**: Latest stable (for Investment service - in development)

## Installation

### Install Bun

```bash
# macOS/Linux
curl -fsSL https://bun.sh/install | bash

# Verify installation
bun --version
```

### Install Go

```bash
# macOS
brew install go

# Verify installation
go version
```

### Install Rust

```bash
# All platforms
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Verify installation
rustc --version
cargo --version
```

### Install Docker

Download from [docker.com](https://www.docker.com/products/docker-desktop)

## Getting Started

### 1. Clone Repository

```bash
git clone https://github.com/your-org/tagaroa.git
cd tagaroa
```

### 2. Install Dependencies

```bash
# Install all dependencies
bun install

# Or using make
make install-deps
```

### 3. Setup Environment

```bash
# Copy environment files
make env-setup

# Or manually
cp apps/auth/.env.example apps/auth/.env
cp servers/auth/.env.example servers/auth/.env
# ... repeat for other services
```

### 4. Start Infrastructure

```bash
# Start Docker services
make docker-up

# Initialize databases
make db-init
```

### 5. Run Development Servers

```bash
# Run all frontend apps
bun run dev

# Or run specific app
turbo dev --filter=web
turbo dev --filter=auth
turbo dev --filter=admin
turbo dev --filter=finance
turbo dev --filter=docs

# Run backend servers (in separate terminals)
cd servers/auth && bun run dev           # TypeScript/Bun
cd servers/storage && bun run dev        # TypeScript/Bun
cd servers/finance && go run ./cmd/main.go  # Go
cd file && go run ./cmd/main.go          # Go
```

## Project Structure

```
tagaroa/
├── apps/              # Frontend applications
│   ├── web/          # Main website (Port 3000)
│   ├── auth/         # Auth app (Port 3001)
│   ├── admin/        # Admin dashboard (Port 3002)
│   ├── finance/      # Finance app (Port 3004)
│   ├── investment/   # Investment app (Port 3005)
│   └── docs/         # Documentation (Port 3010)
├── servers/          # Backend services
│   ├── auth/         # Auth service (TypeScript/Bun, Port 8081)
│   ├── finance/      # Finance service (Go, Port 8082)
│   ├── storage/      # Storage service (TypeScript/Bun, Port 8084)
│   └── investment/   # Investment service (Rust, Port 8083) - In Development
├── packages/         # Shared packages
│   ├── ui/           # UI components (Radix UI, Tailwind)
│   ├── common/       # Common utilities, hooks, stores
│   ├── shared/       # Shared code (Go, Hono, Rust, Proto)
│   ├── posthog-config/  # PostHog analytics config
│   ├── sentry-config/   # Sentry error tracking config
│   └── typescript-config/ # Shared TypeScript configs
├── infra/            # Infrastructure configs
│   ├── k8s/         # Kubernetes manifests with Flux
│   └── terraform/   # Terraform configs
└── Makefile         # Build commands
```

## Development Workflow

### 1. Create Feature Branch

```bash
git checkout -b feature/my-feature
```

### 2. Make Changes

Edit files in your preferred editor.

### 3. Run Linting

```bash
# Lint all code
make lint

# Or specific workspace
turbo lint --filter=web
```

### 4. Run Tests

```bash
# Run all tests
make test

# Run specific tests
cd servers/auth && bun test
cd servers/finance && go test ./...
```

### 5. Build

```bash
# Build all
make build

# Build specific app
turbo build --filter=web

# Build specific server
make build-server SERVER=finance
```

### 6. Commit Changes

```bash
git add .
git commit -m "feat: add new feature"
```

### 7. Push and Create PR

```bash
git push origin feature/my-feature
```

## Turborepo

### Running Tasks

```bash
# Run task for all workspaces
turbo dev
turbo build
turbo lint
turbo test

# Run task for specific workspace
turbo dev --filter=web
turbo build --filter=@repo/common

# Run task for workspace and dependencies
turbo build --filter=web...
```

### Caching

Turborepo caches task outputs:

```bash
# Clear cache
turbo clean

# Force run without cache
turbo build --force
```

### Parallel Execution

Turborepo runs tasks in parallel when possible:

```json
{
  "tasks": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": [".next/**"]
    }
  }
}
```

## Code Style

### TypeScript/JavaScript

Uses Biome for formatting and linting:

```bash
# Format code
bun run format

# Check code
bun run lint

# Fix issues
biome check --write
```

### Go

Uses standard Go formatting:

```bash
# Format code
go fmt ./...

# Lint code
go vet ./...
```

### Rust

Uses rustfmt and clippy:

```bash
# Format code
cargo fmt

# Lint code
cargo clippy
```

## Database Management

### Migrations

#### Auth Server (Drizzle)

```bash
cd servers/auth

# Generate migration
bunx drizzle-kit generate

# Run migration
bunx drizzle-kit migrate

# Studio (GUI)
bunx drizzle-kit studio
```

#### Finance Server (golang-migrate)

```bash
cd servers/finance

# Create migration
migrate create -ext sql -dir db/migrations -seq migration_name

# Run migrations
migrate -path db/migrations -database "postgresql://..." up

# Rollback
migrate -path db/migrations -database "postgresql://..." down
```

### Database Access

```bash
# Connect to PostgreSQL
docker compose exec postgres psql -U postgres

# List databases
\l

# Connect to database
\c auth

# List tables
\dt

# Query
SELECT * FROM users;
```

## Common Tasks

### Add New Dependency

```bash
# Add to specific workspace
cd apps/web
bun add package-name

# Add to root
bun add -D package-name
```

### Create New Package

```bash
# Create directory
mkdir -p packages/my-package/src

# Create package.json
cat > packages/my-package/package.json << EOF
{
  "name": "@repo/my-package",
  "version": "0.0.0",
  "private": true,
  "main": "./src/index.ts"
}
EOF

# Install dependencies
bun install
```

### Add New App

```bash
# Create Next.js app
cd apps
bunx create-next-app@latest my-app

# Update package.json name
# "name": "my-app"

# Add to turbo.json if needed
```

### Add New Server

```bash
# TypeScript server
mkdir -p servers/my-server/src
cd servers/my-server
bun init

# Go server
mkdir -p servers/my-server/cmd
cd servers/my-server
go mod init github.com/your-org/tagaroa/servers/my-server

# Rust server
cargo new servers/my-server
```

## Troubleshooting

### Port Already in Use

```bash
# Find process using port
lsof -i :3000

# Kill process
kill -9 <PID>
```

### Dependencies Not Installing

```bash
# Clear cache
rm -rf node_modules
rm bun.lock

# Reinstall
bun install
```

### Database Connection Issues

```bash
# Check PostgreSQL is running
docker compose ps postgres

# Restart PostgreSQL
docker compose restart postgres

# Check connection
docker compose exec postgres psql -U postgres -c "SELECT 1"
```

### Build Failures

```bash
# Clear Turborepo cache
turbo clean

# Clear Next.js cache
rm -rf apps/*/.next

# Rebuild
turbo build --force
```

## Best Practices

1. **Use Turborepo**: Leverage caching and parallel execution
2. **Write Tests**: Test new features and bug fixes
3. **Type Everything**: Use TypeScript types
4. **Follow Conventions**: Consistent code style
5. **Small Commits**: Atomic, focused commits
6. **Document Changes**: Update docs with code changes
7. **Review PRs**: Thorough code reviews
8. **Run Locally**: Test changes locally before pushing
