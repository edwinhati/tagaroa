# Common Package

Shared utilities, hooks, components, and types used across all applications in the monorepo.

## Overview

The common package provides:
- Shared React components
- Custom hooks
- Authentication utilities
- HTTP client
- State management
- TypeScript types
- Query utilities

## Structure

```
packages/common/
├── src/
│   ├── components/      # Shared components
│   ├── hooks/           # Custom hooks
│   ├── lib/             # Utilities
│   ├── providers/       # Context providers
│   ├── stores/          # Zustand stores
│   └── types/           # TypeScript types
└── package.json
```

## Usage Examples

### Complete Page Example

```typescript
import { LayoutShell } from '@repo/common/components/layout-shell';
import { Loading } from '@repo/common/components/loading';
import { useQuery } from '@tanstack/react-query';
import { httpClient } from '@repo/common/lib/http';
import type { Account } from '@repo/common/types/account';

export default function AccountsPage() {
  const { data: accounts, isLoading } = useQuery({
    queryKey: ['accounts'],
    queryFn: () => httpClient.get<Account[]>('/api/accounts'),
  });

  if (isLoading) return <Loading />;

  return (
    <LayoutShell>
      <div>
        <h1>Accounts</h1>
        {accounts?.map(account => (
          <div key={account.id}>{account.name}</div>
        ))}
      </div>
    </LayoutShell>
  );
}
```

### Authentication Example

```typescript
import { authClient } from '@repo/common/lib/auth-client';
import { redirectToDashboard } from '@repo/common/lib/redirect';
import { toast } from 'sonner';

export async function handleSignIn(email: string, password: string) {
  try {
    await authClient.signIn.email({ email, password });
    toast.success('Signed in successfully');
    redirectToDashboard();
  } catch (error) {
    toast.error('Failed to sign in');
  }
}
```

## Best Practices

1. **Use Typed Imports**: Import types separately
2. **Leverage Hooks**: Use provided hooks for common functionality
3. **Consistent State**: Use Zustand stores for global state
4. **Type Safety**: Always use TypeScript types
5. **Error Handling**: Handle errors consistently
6. **Loading States**: Use Loading component
7. **Authentication**: Use authClient for all auth operations
